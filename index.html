Шпаргалка з командами Git

HW #1 "VCS (Git, Github)" - Viacheslav Pliusnin

Base

  • git config --global user.name "[ім'я]" - встановлює ім'я для ідентифікації автора;
  • git config --global user.email "[електронна адреса]" - встановлює електронну адресу для ідентифікації автора;
  • git init - ініціює пустий репозиторій в поточній директорії;
  • git status - показує стан файлів в робочій директорії;
  • git add - додає файли в коміт. Використовується "." або "*" для додавання усіх файлів. Якщо додається один файл - вказується його ім'я та розширення;
  • git commit - зберігає поточний зліпок проекту. Зазвичай вказується з аргументом -m "[коментар з описом змін]";
  • git log - використовується для перегляду історії комітів поточної гілки проекту;
  • git checkout - використовується для зміни поточної гілки або для відновлення історичної версії файлу;
  • git branch - команда використовується для перегляду, створення або видалення гілок;
  • git push - відправляємо в репозиторій наші зліпки проекту;
  • git pull - команда для отримання змін з репозиторію;
  • git megre - відповідає за об'єднання гілок проекту;
  • git clone - копіює репозиторій в нову директорію.

.gitignore - файл потрібний для того, щоб сховати файли і папки від системи контролю версій Git.

Advanced

  • git revert - за допомогою цієї команди Git створює новий коміт з відміною останнього коміта;
  • git reset - скидання історії комітів до вибранного коміта;
  • git rebase - заміна одного коміта в основі гілки на інший, в результаті чого гілка отримує новий початок;
  • git cherry-pick - використовується для того, щоб взяти зміни, внесені будь-яким комітом і постаратись застосувати їх у вигляді нового коміта нагорі поточної гілки;
  • git stash - поміщає поточні зміни, які виконувались у файлах, в окреме сховище та повертає файли у вихідне положення.
